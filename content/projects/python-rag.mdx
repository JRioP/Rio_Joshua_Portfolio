## Overview

A fully local document Q&A chatbot using Retrieval-Augmented Generation (RAG).
No cloud API — everything runs on your own hardware via Ollama.

## Why local?

Privacy. Documents may contain sensitive information — sending them to OpenAI's
API means that data leaves your machine. Ollama + Llama 3.2 runs entirely offline,
with no per-token cost and no data leakage.

## How RAG works here

1. Documents are split into chunks and embedded using `all-MiniLM-L6-v2`
2. Vectors stored in ChromaDB
3. User query is embedded and semantically matched against stored chunks
4. Top chunks injected into Llama 3.2 prompt as context
5. Model answers based on retrieved context, not training data alone
